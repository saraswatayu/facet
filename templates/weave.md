# Cross-Persona Weaving Instructions

You are weaving social connections between independently-generated personas in a behavioral simulation study.

The study plan (with cross-reference plan) has been provided as a separate file — you should have already read it before reading this template.

---

## Your Task

Read ALL persona files in the personas directory. Then identify and create cross-persona connections.

### What to Look For

1. **Geographic proximity** — Personas in the same city or metro area who could realistically know each other
2. **Professional connections** — Same industry, same company type, same professional community
3. **Family/social ties** — The plan's cross-reference section identifies planned connections
4. **Community overlap** — Same religious community, school, neighborhood, interest group
5. **Referral chain opportunities** — Tech-savvy person who could introduce a non-tech person; organizer who could bring a group

### How to Weave

For each connection you identify:

1. Read both personas carefully
2. Determine which persona would discover the product FIRST (based on their discovery section)
3. Write a REFERRAL ANNOTATION at the end of the LATER persona's DISCOVERY section:

```markdown
**Cross-reference:** [First persona name] (Persona [NNN]) recommended this product to [this persona].
[1-3 sentences describing the specific conversation or moment of referral, using both personas' established communication styles and relationship dynamics.]
```

4. Add a REFERRAL NOTE to the FIRST persona's referral behavior section:

```markdown
**Downstream referral:** Tells [later persona name] (Persona [NNN]) about the product. [1 sentence about how and why.]
```

### Rules

- Only create connections that are REALISTIC given both personas' profiles
- Don't force connections — if two personas in the same city have no plausible social link, skip it
- The referral conversation must feel natural — use each persona's established voice and communication style
- Aim for 8-15 cross-references across the full study (not every persona needs one)
- Prioritize connections that create CHAINS (A→B→C) over isolated pairs
- Use the Edit tool to modify existing persona files — do NOT rewrite them entirely

### Priority Order

1. Connections identified in the plan's cross-reference section (mandatory)
2. Same-city professional connections (if plausible)
3. Family/community connections you notice
4. Serendipitous connections that emerge from reading the personas
